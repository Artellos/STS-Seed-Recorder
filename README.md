# STS Seed Recorder

> **Vibecoded** — this project was built with vibes, caffeine, and AI assistance.

A webapp for recording and visualising all map nodes in **Slay the Spire 2** for a given run seed.

Maps can be generated instantly from a seed string using a pure-Python port of the STS2 map generation algorithm — no game installation required.

---

## Features

- **Seed management** — create, name, and delete seeds
- **Auto-generate maps** — derive all 3 act maps from a seed in milliseconds (pure Python, no STS2 install needed)
- **Manual map editor** — click to place/edit nodes, draw connections, mark your chosen path
- **Node detail panel** — add notes to any node
- **Act tabs** — switch between Act 1, 2, and 3 views
- **Keyboard shortcuts** — `M` Monster, `E` Elite, `R` Rest, `S` Shop, `V` Event, `T` Treasure, `B` Boss, `A` Ancient, `U` Unknown, `C` toggle Connect Mode, `Del` delete selected node
- **Context menu** — right-click any node to change its type, toggle on-path, or delete it
- **Dark STS-themed UI**
- Persistent SQLite storage — no external database needed

## Map Generation

Maps are generated by `backend/map_generator.py`, a faithful Python port of the decompiled STS2 C# source (`MegaCrit.Sts2.Core`).

### Seed → Map pipeline

1. `seed_string` → `GetDeterministicHashCode(seed_string)` → `uint base_seed`
2. For each act *N*: `map_rng = Rng(base_seed, "act_N_map")`
3. `StandardActMap` algorithm:
   - **Path generation** — 7 iterations of random-walk from row 1 to the boss row, no crossing paths
   - **Type assignment** — guaranteed rows (RestSite at top, Treasure/Elite at row−7, Monster at row 1), then queue-based assignment with adjacency constraints
   - **Pruning** — duplicate path segments removed
   - **Post-processing** — center grid, spread adjacent nodes, straighten paths

### Act configurations

| Act | Rooms | Rest count |
|-----|-------|------------|
| Overgrowth (Act 1) | 15 | Gaussian(7, σ1, 6–7) |
| Underdocks (Alt Act 1) | 15 | Gaussian(7, σ1, 6–7) |
| Hive (Act 2) | 14 | Gaussian(6, σ1, 6–7) |
| Glory (Act 3) | 13 | Uniform(5–6) |

### Node types

Auto-generated: `Ancient` · `Monster` · `Elite` · `Treasure` · `Rest` · `Shop` · `Unknown` · `Boss`

Manual placement only: `Event`

## REST API

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/api/seeds` | List all seeds |
| `POST` | `/api/seeds` | Create seed (`seed_value`, `name`) |
| `GET` | `/api/seeds/:id` | Get seed with nodes & connections |
| `PUT` | `/api/seeds/:id` | Update seed |
| `DELETE` | `/api/seeds/:id` | Delete seed |
| `POST` | `/api/seeds/:id/generate` | Generate all act maps from seed (synchronous) |
| `GET` | `/api/seeds/:id/generate/status` | Always returns 100% — retained for compatibility |
| `GET` | `/api/sts2/status` | Check generator availability |
| `POST` | `/api/seeds/:id/nodes` | Add a node manually |
| `PUT` | `/api/nodes/:id` | Update node (type, notes, on_path) |
| `DELETE` | `/api/nodes/:id` | Delete node |
| `POST` | `/api/seeds/:id/connections` | Add a connection |
| `DELETE` | `/api/connections/:id` | Delete connection |
